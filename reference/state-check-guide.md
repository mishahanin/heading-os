# State Check Guide

> The operational rhythm for assessing and maintaining states across the 31C Tribe. Part of the 31C Operational State Model.
> Last Updated: 2026-04-18
> Last Verified: 2026-04-21

---

## What Is a State Check?

A State Check is a diagnostic assessment of what operational state the Tribe — or a function within it — is currently operating in. It replaces traditional goal reviews, status updates, and progress reports.

A State Check does not ask: "Are we on track to hit the number?"
A State Check asks: "What state are we in, and what does the sea demand?"

The vocabulary matters. "Track" implies a rail — fixed, predetermined, inflexible. "State" implies a condition — dynamic, readable, adjustable. The words you use shape the thinking you do.

---

## What a State Check Is NOT

Be clear about this. The boundaries matter as much as the format.

- **Not a status report.** Status reports describe what happened. State Checks diagnose what condition you're in.
- **Not a performance review.** No one is being evaluated. The state is being assessed. Blame has no place here.
- **Not a goal-tracking meeting.** There are no scorecards, no green/yellow/red traffic lights, no "percent to target" slides.
- **Not a place to assign blame for missed targets.** If the state has drifted, the question is "what do we adjust?" not "who is responsible?"
- **Not optional when things are going well.** Drift is most dangerous when it's invisible. The calmest seas still require navigation.

---

## The Four Questions

Every State Check answers four questions. In this order. No more, no less.

### 1. Sea State (2 minutes)

**"What external conditions have changed?"**

What has shifted in the environment since the last check? Market movements, competitor actions, regulatory changes, partner dynamics, client developments, geopolitical shifts. You don't control the sea state — but you must read it to navigate.

Keep this brief. The sea state is context, not the main event.

### 2. Current State (3 minutes)

**"What state are we actually in right now?"**

Not what state you wish you were in. Not what state the plan says you should be in. What state are you *actually* in? Evidence, not opinion.

Reference the behavioral indicators from your Condition Framework. Which indicators are active? Which have gone quiet? Be honest — a State Check that lies to itself is worse than no State Check at all.

### 3. Heading Check (2 minutes)

**"Are we on heading? If not, what drifted?"**

Is the current operational state aligned with where the Tribe / function needs to be? If drift has occurred, name it specifically. Drift is always specific — "things feel off" is not a heading check. "We've started prioritizing release dates over capability thresholds" is.

### 4. Course Corrections (3 minutes)

**"What specific adjustments restore or advance the state?"**

Not "what's the recovery plan for the missed target." What behavior, rhythm, or condition needs adjusting? Course corrections are operational — they change how you work, not what number you chase.

Every course correction should be:
- **Specific** — "Reconnect engineering to deployment environments this week" not "improve product quality"
- **Behavioral** — about what people do differently, not what outcome they achieve
- **Immediate** — actionable within the next State Check cycle, not "by end of quarter"

## Format

### Duration

**15 minutes maximum.** If a State Check takes longer than 15 minutes, it has drifted into a traditional review. Stop. Reset. A systems check on a vessel doesn't take all morning.

### Structure

| Phase | Time | Question | Output |
|-------|------|----------|--------|
| Sea State | 2 min | What external conditions changed? | Shared awareness |
| Current State | 3 min | What state are we actually in? | Honest diagnosis |
| Heading Check | 2 min | Are we on heading? What drifted? | Named drift (if any) |
| Course Corrections | 3 min | What adjustments are needed? | Specific actions |
| Buffer | 5 min | Clarifications, assignments | Clear ownership |

### Rules

- **No slides.** If you need a deck to describe your state, you don't know your state.
- **No scorecards.** Numbers may be referenced as evidence of state, but they are not the subject.
- **Evidence over opinion.** "I think we're doing well" is not a state assessment. "Three of five behavioral indicators are active, two have gone quiet" is.
- **Course corrections, not action items.** Traditional meetings produce action items — tasks to complete. State Checks produce course corrections — adjustments to how you operate.
- **The facilitator asks, doesn't present.** The person running the State Check asks the four questions and holds the time. They don't deliver a report.

## Cadence

| Level | Frequency | Participants | Facilitator |
|-------|-----------|-------------|-------------|
| **Function** | Weekly | Function members | Function lead |
| **Cross-functional** | Biweekly | Function leads | Rotating lead |
| **Tribe-wide** | Monthly | All function leads + Captain | Captain |

### Function-Level (Weekly)

Each function (Sales, Product, Partnerships, etc.) runs its own State Check weekly. This is the most granular and most frequent — drift is caught early at this level.

### Cross-Functional (Biweekly)

Function leads come together to check the state across functions. This is where interdependencies surface: product drift affects sales state, partnership drift affects pipeline state. Cross-functional State Checks prevent siloed drift.

### Tribe-Wide (Monthly)

The Captain reads the sea state across the entire Tribe. This is the navigation bridge — the broadest view, the longest horizon. At this level, the questions expand slightly:
- Sea State: What's changed in our markets, competitive landscape, and strategic environment?
- Current State: What operational state is the Tribe in? What's strong? What's drifting?
- Heading: Are we on heading toward the conditions defined in our Operational State Model?
- Course Corrections: What do I (the Captain) need to adjust? What do function leads need to adjust?

## Side-by-Side: Traditional Review vs. State Check

| Dimension | Traditional Review | State Check |
|-----------|-------------------|-------------|
| **Core question** | "Are we on track to hit the target?" | "What state are we operating in?" |
| **Measures** | Progress toward a number | Quality of operational condition |
| **When targets are missed** | "What's the recovery plan?" | "What condition drifted and what do we adjust?" |
| **When targets are hit** | "Celebrate and set a higher target" | "Confirm the state is sustainable and maintain it" |
| **Vocabulary** | Targets, KPIs, milestones, red/yellow/green | States, heading, sea state, drift, course correction |
| **Failure mode** | Hitting the number but destroying the condition (burning out the Tribe to make quota) | N/A — the state IS the measure, so maintaining it maintains the people |
| **Duration** | 30-60 minutes (often more) | 15 minutes maximum |
| **Output** | Action items, updated forecasts, escalations | Course corrections — behavioral adjustments |
| **Emotional tone** | Anxiety when behind, relief when ahead | Diagnostic neutrality — "what state are we in?" has no judgment |
| **What it rewards** | Hitting numbers regardless of method | Maintaining conditions that produce numbers sustainably |

## Example: Sales Function State Check

**Sea State (2 min):**
"Two developments this week. First, a government RFP in Central Asia that our partner DistributorCo flagged — aligns with sovereign DPI requirements. Second, a competitor announced a new partnership in the Gulf region. Not a direct threat to our pipeline, but worth monitoring."

**Current State (3 min):**
"Checking against our indicators: Inbound inquiries are active — we received three this week without outbound. Partners are bringing pipeline — PartnerCo registered a new opportunity independently. Pipeline depth is at 2.8x, slightly below the 3x threshold. Deal conversations are value-centered, no discounting pressure. Four of five indicators are active. The pipeline depth indicator is approaching drift territory."

**Heading Check (2 min):**
"We're on heading. The pipeline depth softness is regional — Central Asia pipeline is strong, Gulf is rebuilding after our repositioning there. Not systemic drift, but worth watching for one more cycle."

**Course Corrections (3 min):**
"One correction: increase partner enablement in the Gulf region specifically. DistributorCo's dealer network has capacity we haven't activated. Schedule a joint enablement session this week focused on the Gulf pipeline opportunity. If pipeline depth doesn't recover to 3x by next State Check, we escalate to cross-functional."

**Total time: 10 minutes.**

## Getting Started

### If you've never run a State Check:

1. Read the Operational State Model (`outputs/operations/workspace/31c-operational-state-model.md`) — understand the states defined for your function
2. Complete a Condition Framework (`reference/condition-framework-template.md`) for your area — this gives you the behavioral indicators and drift signals you'll reference
3. Schedule your first State Check — keep it to 15 minutes
4. Run through the four questions honestly — the first one will feel awkward, the third one will feel natural
5. End with course corrections, not action items — remember: you're adjusting how you operate, not what you chase

### Common first-time mistakes:

- Spending too long on Sea State (it's context, not the main event)
- Describing the state you want instead of the state you're in
- Producing action items instead of course corrections
- Going over 15 minutes (stop at 15, even if you're not done — brevity is a discipline)
- Skipping the Heading Check because "we're fine" (drift is most dangerous when invisible)
