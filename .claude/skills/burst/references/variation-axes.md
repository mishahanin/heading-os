# Burst - Variation Axes Catalogue

Consumed by: `.claude/skills/burst/SKILL.md` Phase 2 Block B when picking distinct axes for the N-1 spread variants and the mandatory swing variant.

Last Updated: 2026-06-10

## Axis pool for spread variants

Pick distinct axes - never produce two variants on the same axis. If only K-1 spread axes are defensible, pick the next-best axis rather than repeating an axis with different framing.

- **Opener style** - swap the first 1-3 sentences for a different hook
  (hook -> statement, question -> declaration, anecdote -> data point).
- **Tone** - shift the register (analytical -> conversational, founder
  voice -> operator voice, technical -> narrative).
- **Structure** - reorder the spine (problem-solution -> solution-
  problem, three-act -> one-cut, bullet list -> running prose).
- **Lens** - reframe through a different angle (technical -> commercial,
  market -> personal, abstract -> specific case study).
- **Length** - tighten or expand (50% shorter or 2x). Only one length
  variant per pass.
- **Voice** - shift posture (committed -> exploratory, declarative ->
  question-led, internal -> external voice).
- **Metaphor system** - swap the controlling metaphor (maritime ->
  mechanical, financial -> biological, military -> craft).

## Swing-variant rules

- The swing variant is mandatory and explicit. Label it:
  `## Variant {N} - swing: <opposite approach description>`
- The swing inverts a defining property of the original. If long-form
  explanatory, the swing is a one-line provocation. If data-led, the
  swing is anecdote-led. If enthusiastic, the swing is sceptical.
- Even when the swing fails the user's purpose, its presence forces
  explicit perspective-checking. That is its job.

## Sort order

- Variants 1..N-1: spread variants ordered by axis adjacency to the
  original. Closest-to-original first, most-distant-axis last among
  the spreads.
- Variant N: always the swing variant.
