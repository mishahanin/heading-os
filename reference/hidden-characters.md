# Hidden Unicode Characters in AI-Generated Text

> Policy and detection reference for invisible Unicode contamination in generated content.
> Last Updated: 2026-04-18

## The Problem

AI-generated text can contain invisible Unicode characters that don't render visually but exist in the data. These cause real problems: broken code, failed string comparisons, document formatting issues, and third-party AI detection.

## Characters We Strip

| Character | Unicode | Effect |
|-----------|---------|--------|
| Zero-width space | U+200B | Invisible space with no width |
| Zero-width non-joiner | U+200C | Invisible, affects ligature rendering |
| Zero-width joiner | U+200D | Invisible, joins characters |
| Left-to-right mark | U+200E | Invisible directional control |
| Right-to-left mark | U+200F | Invisible directional control |
| Soft hyphen | U+00AD | Invisible unless line breaks there |
| Non-breaking space | U+00A0 | Looks like a space, behaves differently (replaced with regular space) |
| Word joiner | U+2060 | Invisible, prevents line breaks |
| Byte order mark | U+FEFF | Invisible marker (valid only at file start) |
| Invisible times | U+2062 | Invisible mathematical operator |
| Invisible separator | U+2063 | Invisible separator |
| Invisible plus | U+2064 | Invisible mathematical operator |
| Combining grapheme joiner | U+034F | Invisible combining character |
| Arabic letter mark | U+061C | Invisible directional control |
| Mongolian vowel separator | U+180E | Invisible separator |
| Line separator | U+2028 | Non-standard line break |
| Paragraph separator | U+2029 | Non-standard paragraph break |
| Function application | U+2061 | Invisible mathematical operator |

## Why They Appear

1. **Watermarking** -- Some AI providers embed invisible character patterns to fingerprint AI-generated text for detection purposes.
2. **Tokenizer artifacts** -- LLM training data contains these characters; the model reproduces them without awareness.

## Workspace Policy

**All text generated by Claude in this workspace must be free of hidden Unicode characters.** This is a non-negotiable rule in CLAUDE.md.

Claude is instructed to:
- Never intentionally include invisible Unicode characters
- Treat hidden character exclusion as a quality standard on par with factual accuracy
- Flag any uncertainty about output cleanliness

## Sanitizer Tool

**Script:** `scripts/sanitize-text.py`

```bash
# Scan a file for hidden characters (no changes)
python scripts/sanitize-text.py --scan document.md

# Clean a file in place
python scripts/sanitize-text.py document.md

# Clean to a new file
python scripts/sanitize-text.py document.md -o clean-document.md

# Clean from clipboard/stdin
pbpaste | python scripts/sanitize-text.py -
```

Use the sanitizer as a verification step on any high-stakes output (proposals, contracts, code, investor materials) before delivery.
